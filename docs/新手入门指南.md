# 添梯新手入门指南

## 项目定位

添梯是一个以工作流为中心的 AI 创作平台，核心目标是把「剧本 → 人物资产 → 分镜 → 提示词 → 视频生成」整理成可复用、可扩展的固定流程。

- 核心入口：工作流中心
- 核心场景：漫剧工作流
- 运行方式：前端应用 + 本地或远程 API 服务
- 存储方式：浏览器本地存储 + 可选 PostgreSQL

---

## 环境要求

建议使用以下基础环境：

- `Node.js >= 18`
- `npm >= 8` 或 `pnpm >= 8`
- 现代浏览器：`Chrome` / `Edge`
- 可选：`Docker Desktop`（如果要本地运行 PostgreSQL）

---

## 快速启动

### 1. 安装前端依赖

```bash
npm install
```

### 2. 启动前端

```bash
npm run dev
```

默认访问地址：

- `http://localhost:5173`

### 3. 启动本地 API

```bash
npm run server:dev
```

默认接口地址：

- `http://localhost:3001/api`

健康检查：

- `http://localhost:3001/api/health`

---

## 本地数据库模式

如果你希望本地开发直接对接 PostgreSQL，可以使用仓库自带的 `docker-compose.db.yml`。

### 1. 启动数据库容器

```bash
docker compose -f docker-compose.db.yml up -d
```

### 2. 启动服务端

```bash
npm --prefix server install
npm run server:dev
```

默认数据库连接：

- `postgresql://postgres:postgres@localhost:5433/aiyou`

## 工作流使用建议

当前系统推荐从工作流中心进入，而不是在画布里手工拼装大量离散节点。

推荐顺序：

1. 创建项目
2. 进入工作流中心
3. 添加漫剧工作流模板
4. 先产出系列级资产（人物 / 场景 / 道具）
5. 再批量推进剧集、分镜、提示词和视频

这样做的好处：

- 资产可复用
- 节点关系更稳定
- 画布复杂度更低
- 后续扩展新工作流更容易

## 文本编码约定

为了避免再次出现中文乱码，仓库现在统一采用 UTF-8 无 BOM，并提供了两条维护命令：

```bash
# 检查仓库是否存在 BOM、替换符、私有区字符、典型乱码片段
npm run check:text-integrity

# 自动移除 UTF-8 BOM，并报告仍需人工处理的文件
npm run fix:text-encoding
```

同时仓库新增了：

- `.editorconfig`
- `.gitattributes`

这两份文件会一起约束编辑器和 Git 的文本编码行为。

## 常见问题

### 页面能打开，但没有数据

优先检查：

- 服务端是否启动
- `VITE_API_BASE` 是否正确
- 浏览器控制台是否有报错

### 本地数据库连不上

优先检查：

- Docker 是否已启动
- `5433` 端口是否被占用
- `DATABASE_URL` 是否与容器配置一致

### 即梦相关功能异常

优先检查：

- 登录状态是否有效
- 页面结构是否变化
- 即梦当前能力是否仍支持对应入口

## 建议的开发顺序

如果你要继续升级当前项目，建议按下面顺序推进：

1. 先稳定工作流中心
2. 再收敛漫剧工作流模板
3. 然后补强资产复用和批量推进
4. 最后再扩展更多工作流类型

这样可以保证系统结构先稳定，再扩展功能。
