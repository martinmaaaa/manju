# Tianti V2.0 规划蓝图（现状版）

> 状态：当前有效  
> 基线：与 [plans/PLAN.md](C:/Users/marti/Desktop/Code/aiyou/plans/PLAN.md) 保持一致

## V2.0 的核心判断

V2.0 不是继续扩展旧的通用画布，而是把系统彻底收敛成一个围绕项目、剧集、分镜和视频生成的工作流产品。

一句话概括：

**从“节点工具集合”升级成“围绕系列生产的工作流系统”。**

## 当前已经完成的部分

### 1. 产品主入口收口
- 项目流已经成为主入口
- `Canvas Studio` 是独立沙盒，不再主导产品心智

### 2. 模型底座收口
- 新工作流之外的旧模型体系已清理
- 当前模型底座统一维护在：
  - [model-foundation-registry.md](C:/Users/marti/Desktop/Code/aiyou/plans/model-foundation-registry.md)

### 3. 节点级模型与 schema
- 文本、图片、视频节点都支持独立 deployment
- 参数面板按 schema 动态渲染
- 视频节点支持模式、输入槽位和高级设置

### 4. 单集工作台收口
- 底部分镜视频条已经从 beat list 升级成最终结果条
- 右侧栏已经改成资产 / 检查器双 tab
- 主预览、分镜条、异步状态、采用结果的关系已经理顺

### 5. skills schema 启动
- `director`
- `episode`
- `storyboard`
- `asset_extract`
- `image_prompt_generate`

这些核心链路已经开始按 schema 驱动，而不是全部写死在 capability 里。

## 当前还在继续推进的部分

### 1. skills 整体内化
- 继续让更多能力完全依赖 skill schema
- 逐步降低 `capabilityEngine` 中剩余的业务特例

### 2. 即梦执行链稳定性
- 多参模式已实机探明
- 仍需继续提升真实生成、排队、回收结果的稳定性

### 3. 文档与运行时一致性
- `plans` 和 `docs` 已开始统一收口
- 后续所有结构变化都要同步更新基线文档

## 当前版本边界

### 当前应该继续强化
- `Project Workflow`
- 资产版本化
- 分镜结果管理
- 节点级模型能力
- skills schema
- review rule registry

### 当前不再回退的方向
- 旧 `components/workflow` 主中心
- 旧 `services/api` 客户端体系
- 旧 `workflowTemplates`
- 旧全局 provider 模型心智
- 旧根级 `types.ts`

## 成功标准

如果从今天往后看，V2.0 的判断标准不是“还有多少旧代码没删”，而是：

- 新项目是否沿着项目流、剧集流、工作台流顺畅跑通
- 模型、skills、review、shot strip 是否已经是一套统一底座
- 后续再接模型、再接 skill、再调工作台时，是否不需要重新发明旧体系
