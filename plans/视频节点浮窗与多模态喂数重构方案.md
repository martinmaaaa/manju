# 视频节点浮窗与多模态喂数重构方案

## Summary
- 视频节点沿用你给的交互方向：**紧凑节点 + 底部摘要芯片 + 设置浮窗 + 高级设置折叠区**。
- 图片节点那套“参数由模型能力动态显示”的原则继续保留，视频节点也一样，**不把具体模型字段写死在组件里**。
- 第一批统一收口全部现有模型的喂数规则，但能力展示坚持**实测通过再展示**。
- `seedance2-bendi` 先只做一个视频节点形态，生成方式只保留：
  - `首尾帧`
  - `参考`
  - 不填尾帧时，`首尾帧` 自动等价于图生视频
- 参考素材主要通过**画布连线**进入，视频节点自己只显示槽位状态，不做主上传入口。

## Important Interface Changes
- `CanvasConnection`
  - 新增 `inputKey`
  - 连线语义改成 `from -> to.inputKey`
- `CanvasModelInputDefinition`
  - 从单一 `type` 改成 `accepts: CanvasInputValueType[]`
  - 新增 `slotKind: 'frame' | 'reference' | 'analysis' | 'prompt'`
  - 保留 `label / required / multiple / maxItems`
- `ModelDefinition`
  - 保留 `inputSchema / configSchema`
  - 新增 `generationModes?: Array<{ id; label; summaryLabel; enabledInputKeys: string[] }>`
  - 新增 `defaultGenerationModeId?: string`
- `CanvasNode`
  - 新增 `modeId?: string`
  - 视频节点底部摘要由 `modeId + 核心参数` 动态拼接

## Implementation Changes
### 1. 节点浮窗渲染规则
- 视频节点设置浮窗采用固定结构：
  - `生成方式`
  - `常用设置`
  - `高级设置`
- 但里面显示什么字段，仍然来自模型定义：
  - `generationModes` 决定是否显示“生成方式”
  - `configSchema` 决定有哪些参数项
  - `inputSchema` 决定显示哪些输入槽位卡
- 前端只负责渲染，不写死某个模型专属字段面板。

### 2. 常用设置 / 高级设置分组
- 你已定规则：**前端按规则猜，不要求模型声明手工分组**。
- 第一版前端通用分组规则：
  - 常用设置：`generationMode / ratio / resolution / duration / generateAudio`
  - 其他字段默认全部进“高级设置”
- 这是**通用字段名映射**，不是按模型名硬编码。
- 未知新字段一律先进高级设置，不阻塞模型扩展。

### 3. 输入槽位与连线
- 目标节点是否接受输入，只看目标 deployment 的 `inputSchema` 槽位定义。
- 第一版统一连线规则：
  - 文本可连到支持文本槽位的 `text / image / video`
  - 图片可连到支持图片槽位的 `text / image / video`
  - 视频可连到支持视频槽位的 `text / video`
  - 音频可连到支持音频槽位的 `video`
- 连线时如果目标节点有多个可接槽位：
  - 弹出轻量槽位选择
- 如果只有一个可接槽位：
  - 直接连上

### 4. 各 deployment 第一版能力
- `Gemini 3.1 Pro`
  - 先开放：文本输入、图片输入
  - 用于文本生成和图片解析
- `Nano Banana 2`
  - 先开放：文本输入、图片输入
  - 用于图生图和改图
- `Grok Video 3`
  - 先开放：文本输入、图片输入
  - 不显示生成方式
- `seedance2-bendi`
  - 显示生成方式
  - 第一版模式：
    - `首尾帧`
    - `参考`
  - 当前默认只开放已验证槽位：
    - `text`
    - `startFrame`
    - `endFrame`
    - `referenceImages`
  - `referenceVideos / referenceAudios` 先保留在研究范围，不进可见 UI，等你登录后实机验证通过再开

### 5. Runtime 喂数
- `canvasNodeRuntime` 改成按 `inputKey` 收集输入，不再只用 `text/image/video/audio` 大桶。
- `BLTCY` runtime
  - 文本：文本 + 图片
  - 图片：文本 + 图片参考
  - 视频：文本 + 图片参考
- `seedance2-bendi / Jimeng` runtime
  - 按槽位明确映射：
    - `startFrame`
    - `endFrame`
    - `referenceImages`
  - 不再只把所有图片输入混成一个参考数组
- 视频/音频参考只有在实机确认即梦页面真有对应入口、并确认提交方式后，才接入 runtime。

### 6. 研究与验证
- 先整理一份 `plans` 下的视频模型输入矩阵文档：
  - 每个 deployment 当前实测支持的输入槽位
  - 参数项
  - 是否有生成方式
  - 当前 runtime 映射方式
- 之后你登录即梦，我们做一次实机验证：
  - 页面实际有哪些模式
  - 是否真支持视频参考
  - 是否真支持音频参考
  - 各素材上限是多少
  - 提交后结果如何回收
- 实测通过后，再把 `seedance2-bendi` 的视频/音频参考放开。

## Test Plan
- 连线测试
  - `inputKey` 连线校验正确
  - 多槽位目标节点能正确选槽位
  - 不支持的输入类型会被明确拒绝
- UI 测试
  - 视频节点浮窗能按 `generationModes + configSchema + inputSchema` 动态显示
  - 单模式视频模型不显示“生成方式”
  - `seedance2-bendi` 显示 `首尾帧 / 参考`
  - `尾帧` 为空时仍可执行，摘要表现为图生视频
  - 未识别字段自动进入高级设置
- Runtime 测试
  - Gemini 可接图片解析
  - Nano Banana 继续支持图生图
  - Grok Video 3 只接受文本和图片
  - seedance2-bendi 第一版只接受已开放槽位
- 实机测试
  - 你登录即梦后验证真实上传入口和限制
  - 只有验证通过的参考类型才开放到 schema 和 UI

## Assumptions
- 这轮不考虑旧数据兼容，可以直接升级连接结构和输入 schema。
- 视频节点设置浮窗的外观结构固定，但字段内容动态渲染。
- 常用 / 高级分组由前端通用规则决定，不要求每个模型单独声明。
- `seedance2-bendi` 的视频/音频参考当前只作为研究目标，不在本轮默认开放。
